## 단계 1: 데이터 준비 및 데이터로더 구성 (현재 진행 중)
- [x] **데이터 로드:** `pandas`를 사용해 로컬 `./data` 폴더의 NSMC 텍스트 로드
- [x] **전처리 함수화:** 중복 제거, 결측치 제거, `Okt` 토크나이징 및 불용어 제거
- [x] **사전 구축:** 문장을 정수로 인코딩하고 `word_to_index`, 특수토큰 구성
- [x] **디코딩 및 검증:** `index_to_word` 역방향 딕셔너리를 생성하여 변환된 데이터가 제대로 원래 문장으로 잘 디코딩되는지 작동 검증 및 확인

## 단계 2: 모델 구성을 위한 데이터 분석 및 가공
- [x] **문장 길이 분포 파악:** `X_train` 및 `X_test` 병합 후 문장 길이의 평균, 최소, 최대, 표준편차 계산
- [x] **최대 길이(`maxlen`) 적절하게 설정:** (평균 + 2 * 표준편차) 공식을 적용하여 문장의 약 95% 이상을 보존할 수 있는 `maxlen` 값 도출
- [x] **문장 길이 맞추기(Padding):** 짧은 문장은 `0(<PAD>)`으로 채우고 긴 문장은 잘라내어 모든 행렬 길이를 균일하게 맞추고 PyTorch `Tensor` 구조로 변환

## 단계 3: 모델 구성 및 Validation(검증용) 데이터 스플릿
- [x] **검증 데이터 분리:** 학습 데이터(`X_train`) 내에서 모델 최적화를 평가하기 위해 임의의 20%~30% 데이터를 Validation Set으로 떼어내기 (`train_test_split` 등 활용)
- [x] **3가지 종류 이상의 감성분석 모델 클래스 구축 (`nn.Module`):**
    1. **RNN(LSTM) 아키텍처:** 순차 처리에 특화된 자연어 최고 효율 모델
    2. **CNN(1D Convolution) 아키텍처:** 문장 전체의 로컬 문맥 피처 추출에 효율적인 모델
    3. **GlobalMaxPooling1D 아키텍처:** 시퀀스 전체에서 가장 강력한 하나의 특징을 뽑는 단순하고도 빠른 모델

## 단계 4: 모델 훈련 및 시각화 평가
- [x] PyTorch의 `DataLoader`와 `TensorDataset`을 구축하여 배치(Batch) 단위 환경 조성
- [x] Loss(`BCELoss`), Optimizer(`Adam`) 정의
- [x] 각 모델 별로 훈련 에포크 단위로 Loss, Accuracy 계산 및 출력
- [x] **시각화 (Matplotlib):** Train/Val Loss 및 Accuracy 그래프를 그려보고 최적의 조기종료(Epoch) 시점이 어디인지 분석. 모델 테스트 성능 비교 분석.

## 단계 5: 자체 학습된 임베딩 레이어의 한계 분석
- [x] 가장 성능이 높았던 모델을 꼽아 해당 모델의 `Embedding` 파라미터 벡터를 `gensim` 포맷 텍스트로 로컬 저장
- [x] `gensim.models.keyedvectors`를 통해 임의의 한국어 단어(예: '감동', '사랑')를 주고 이를 얼마나 유의미하게 묶어주는지 자체 임베딩의 성능 측정

## 단계 6: 한국어 사전학습 Word2Vec 임베딩 전이학습 (성능 부스팅)
- [x] `./data/word2vec_ko.model` (사전 훈련된 한국어 Word2Vec 모델) 로드. 
    * _※주의사항:_ `gensim.models.Word2Vec.load()` 메소드로 파일을 읽고 `.wv` 속성으로 접근
- [x] **전이 학습:** 기존 모델에 넣기 위한 빈 행렬 테이블을 만들고 방대한 Word2Vec 사전에 존재하는 한글 단어일 경우 사전학습 벡터 값을 그대로 카피(복붙)
- [x] 복사된 고성능 임베딩 파라미터로 대체된 동일한 딥러닝 모델 구성
- [ ] **재학습 및 성능 폭발 검증:** 새로운 임베딩으로 모델을 학습시키고, Test Accuracy가 이전 깡통 임베딩 모델보다 훨씬 유의미하게(가급적 기준점인 85% 이상) 상승했는지 확인하여 쥬피터 노트북 실습 성공적으로 마무리!
