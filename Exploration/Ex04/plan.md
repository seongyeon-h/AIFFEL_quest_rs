# 🚀 WEAT Project 구현 계획서 (`WEAT_project.ipynb`)

본 계획서는 영화 시놉시스 코퍼스를 활용하여 모든 장르(21개) 간의 편향성을 측정하는 WEAT 점수를 계산하고 시각화하는 프로젝트의 4단계 가이드라인입니다.

## 📋 문서 작성 및 코드 구현 규칙 (Ground Rules)
1. **보고서 형식의 어조 유지:** 마크다운에 기재되는 모든 설명과 결론은 공식적인 프로젝트 보고서 양식에 알맞은 객관적이고 명확한 어조로 작성한다.
2. **과장된 표현 지양:** 불필요하게 감정적이거나 과장된 형용사 표현은 배제하고, 사실 기반의 건조하고 담백한 문장으로 서술한다.
3. **상세한 코드 주석:** 구현되는 모든 코드 블록과 셀에는 각 라인 혹은 블록 단위의 동작 원리를 명확하게 설명하는 상세한 주석을 의무적으로 포함한다.

---

## 📌 STEP 1. 형태소 분석기를 이용하여 품사가 명사인 경우 해당 단어 추출하기

### 📝 마크다운 기획
- **목표 설명:** 방대한 영화 시놉시스 텍스트에서 단어의 의미적 편향성을 제대로 파악하기 위해, 무의미한 조사나 어미를 배제하고 핵심 의미를 지닌 **'명사'**만을 추출하는 이유를 설명합니다.
- **도구 소개:** `konlpy`의 `Okt` 형태소 분석기를 사용한 전처리 과정임을 명시합니다.

### 💻 코드 구현 기획
1. **데이터 로드:** 예술영화(`synopsis_art.txt`), 일반영화(`synopsis_gen.txt`) 및 21개 장르 데이터 파일들을 `os` 모듈을 사용해 불러옵니다. (단, 윈도우 환경이므로 `open(..., encoding='utf-8')` 필수 적용)
2. **명사 추출 (`Okt`):** `okt.pos()`를 이용해 품사가 `"Noun"`인 단어만 `tokenized` 리스트에 저장합니다.
3. **백업 로직 (선택/권장):** 파싱에 약 15~20분이 소요되므로, 커널 재시작 대비용으로 `pickle`을 활용해 추출된 명사 리스트를 `.pkl` 파일로 로컬에 백업하는 코드를 추가합니다.

---

## 📌 STEP 2. 추출된 결과로 Embedding Model 만들기

### 📝 마크다운 기획
- **목표 설명:** 추출된 명사 코퍼스를 사용하여 컴퓨터가 계산할 수 있는 형태의 **밀집 벡터(Dense Vector)** 공간에 단어들을 배치(임베딩)하는 과정을 설명합니다.

### 💻 코드 구현 기획
1. **Word2Vec 모델 훈련:** `gensim.models.Word2Vec`을 불러와 앞서 얻은 `tokenized` 데이터를 훈련시킵니다.
    - 하이퍼파라미터 예시: `vector_size=100`, `window=5`, `min_count=3`, `sg=0` (CBOW)
2. **모델 저장:** 훈련된 모델 객체를 저장하여 추후 단계에서 재사용 가능하게 구성합니다.
    - `model.wv.save_word2vec_format('./weat/data/word2vec.txt')` 

---

## 📌 STEP 3. Target, Attribute 단어 셋 만들기 (🔥 핵심 미션)

### 📝 마크다운 기획
- **기존 TF-IDF의 한계 지적:** "사랑", "영화" 같이 어느 장르에서나 자주 쓰이는 단어가 여러 장르에서 동시에 상위권 키워드로 잡히는 현상(중복 발생)이 WEAT 점수를 희석시킨다는 문제점을 짚어냅니다.
- **개선 방법 제시:** 이를 해결하기 위해 각 장르의 상위 단어들을 뽑은 뒤, **서로 중복되는 단어(교집합)를 제거하고 해당 장르만의 고유한 키워드만 남기는 전략**을 설명합니다.

### 💻 코드 구현 기획
1. **TF-IDF 행렬 구성:** `TfidfVectorizer`를 이용해 Target(예술/일반) 및 Attribute(21개 장르)의 어휘 점수를 계산합니다.
2. **고유 대표 단어 추출 알고리즘:**
    - 각 장르별로 TF-IDF 점수가 높은 상위 $N$개(예: 30~50개)의 단어를 뽑아 리스트(또는 Set)로 만듭니다.
    - **중복 제거 로직:** 하나의 단어가 두 개 이상의 장르 상위 목록에 존재한다면, 가장 점수가 높은 장르에만 남기거나 혹은 아예 모든 장르에서 해당 단어를 삭제해버립니다. (파이썬의 `set` 연산자 활용)
3. **결과 확인:** 중복이 제거된 각 장르별 고유 대표 단어(Attribute 단어 셋)를 출력하여 확인합니다.

---

## 📌 STEP 4. WEAT Score 계산과 시각화

### 📝 마크다운 기획
- **WEAT 원리 요약:** 완성된 고유 단어 셋을 기반으로 장르 간의 유사도 편향성(WEAT 스코어)을 표(Heatmap)로 그려내어 한눈에 인사이트를 도출하는 단계임을 설명합니다.
- **결과 해석 란:** 출력된 21x21 히트맵을 보고, "가장 예술영화와 가까운 장르는 무엇인지", "가장 일반영화와 가까운 장르는 무엇인지" 등의 분석 내용을 가이드에 맞게 서술합니다.

### 💻 코드 구현 기획
1. **WEAT 함수 정의:** 두 Target 셋과 두 Attribute 셋 간의 거리를 계산하는 수학 수식(코사인 유사도 기반)을 파이썬 함수로 구현합니다.
2. **21x21 매트릭스 계산:** `21 x 21` 크기의 빈 리스트를 만들고, 이중 `for` 루프를 순회하며 21종류 장르들 간의 1:1 WEAT 점수를 계산하여 표에 채웁니다.
3. **시각화 (Heatmap):** 
    - `seaborn.heatmap`을 활용하여 색상으로 직관적인 비교가 가능하게 합니다.
    - **주의:** 윈도우 한글 깨짐 방지를 위해 `plt.rc('font', family='Malgun Gothic')`와 마이너스 깨짐 방지 코드를 반드시 먼저 실행합니다.
